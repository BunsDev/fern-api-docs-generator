---
title: Generate your Webhook Reference
description: Use Fern Docs to generate Webhook Reference documentation from your API definition, using either OpenAPI or Fern Definition.
---

Fern can generate Webhook Reference documentation from your API definition. 

<Steps>

    <Step>
        * If you're using an OpenAPI specification with version 3.1+, add a `webhooks` field to the root level of the file.

        ```yml title="openapi.yaml"
        webhooks:
          newPet:
            post:
              requestBody:
                description: Information about a new pet
                  content:
                    application/json:
                      schema:
                        $ref: "#/components/schemas/Pet"
              responses:
                "200":
                  description: webhook processed successfully
        ```

        * If you're using an OpenAPI specification with a version before 3.1, add an [x-fern-webhook extension](/learn/api-definition/openapi/webhooks) to any defined webhooks in that file.

        ```yml title="openapi.yaml"
        TODO snippet goes here
        ```

        * If you're using a Fern Definition, add `webhooks` to any defined webhooks in that file.

        ```yml title="todo-fix-title.yaml"
        TODO snippet goes here
        ```

    </Step>

    <Step>
        * In `docs.yml`, add an `- api: Webhook Reference` entry under `navigation`

        ```yml title="docs.yml" {11-12}
        navigation:
        - section: Introduction
        contents:
        - page: Getting Started
        path: ../introduction/getting-started.md
        - page: Authentication
        path: ../introduction/authentication.md
        - api: API Reference
        api-name: api-v1
        display-errors: true
        - api: Webhook Reference
        api-name: webhooks-v1
        ```
    </Step>
</Steps>

For an example of webhook documentation generated from an API definition, see [Webflow's Webhooks](https://developers.webflow.com/data/reference/webhooks/events/form-submission).

### Directory structure

Webhooks should be defined in a dedicated folder within your Fern project.

<Tabs>
<Tab title="OpenAPI">
```bash
fern/
  └── apis/
      ├── webhooks-v1/           # Webhook definition
      │   ├── openapi/       
      │   │   └── openapi.yml
      │   └── generators.yml
      └── api-v1/                # Regular API endpoints
```

If you're using OpenAPI, `generators.yml` must point to your OpenAPI specification.

```yml generators.yml
api:
  path: openapi/openapi.yml
```

Read more about [defining webhooks](/learn/api-definition/openapi/webhooks) in your OpenAPI specification.

</Tab>
<Tab title="Fern Definition">
```bash
fern/
  └── apis/
      ├── webhooks-v1/           # Webhook definition
      │   ├── definition/  
      │   │   ├── api.yml    
      │   │   └── webhooks.yml
      │   └── generators.yml
      └── api-v1/                # Regular API endpoints
```

Read more about [defining webhooks](/learn/api-definition/fern/webhooks) in your Fern Definition [here].

</Tab>
</Tabs>

### Include more than one Webhook Reference

To include multiple sets of webhook definitions in your documentation, use the `api-name` property:

```yaml title="docs.yml"
navigation:
  - api: Payment Webhooks
    api-name: payment-webhooks
  - api: Order Webhooks
    api-name: order-webhooks
```

When using multiple webhook definitions, organize them in separate directories within your Fern project:

```bash
fern/
  └── apis/
      ├── payment-webhooks/     # Payment webhook definitions
      │   ├── openapi/
      │   │   └── openapi.yml   # Payment webhook OpenAPI spec
      │   └── generators.yml
      └── order-webhooks/       # Order webhook definitions
          ├── openapi/
          │   └── openapi.yml   # Order webhook OpenAPI spec
          └── generators.yml
```
