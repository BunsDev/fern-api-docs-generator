## Setup

HTTP snippets are enabled by default for all Fern documentation sites. If you wish to disable them, you can do so in your `docs.yml` configuration file:

<CodeBlock title="docs.yml">
```yaml 
navigation:
  - api: API Reference
    snippets:
      sdk: # SDK snippets configuration
        typescript: your-package-name    # Enable TypeScript SDK snippets
        python: your-package-name        # Enable Python SDK snippets
      http: # HTTP snippet configuration
        targets:          # order determines display order in dropdown
          - id: python    # default: requests
          - id: php
            library: guzzle   # override default library
          - id: curl
          - id: javascript
            library: axios    # override default fetch
          - id: java     # default: okhttp
          - id: go       # default: native
          - id: ruby     # default: native
          - id: csharp   # default: restsharp
          
        # Or disable HTTP snippets entirely
        enabled: false    # defaults to true

default-language: typescript
```
</CodeBlock>

## How It Works

### Request Examples

To generate HTTP snippets, add request examples to your API definition:
- For Fern Definition: Follow the [examples documentation](/learn/api-definition/fern/examples)
- For OpenAPI: Follow the [request/response examples documentation](/learn/api-definition/openapi/extensions/others#request--response-examples)

### Generated Features

HTTP snippets automatically include:
- Authentication headers with placeholders (e.g., `<apiKey>`)
- Query parameters and request body formatting
- Content-Type headers
- Error handling patterns
- SSL/TLS configuration where applicable

### Display Behavior

- If your API has SDK snippets, those will be shown by default
- If no SDK snippets exist, HTTP snippets will display automatically
- User language preferences are saved client-side
